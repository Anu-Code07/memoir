# Configuration

Memoir uses a hierarchical configuration system managed by the CLI. Configuration is minimal by default but offers extensive customization when needed.

<Callout type="info" title="Memoir Cloud Coming Soon">
  This guide covers self-hosted configuration. **Memoir Cloud** will offer zero-config managed deployments—coming soon.
</Callout>

---

## Configuration Hierarchy

The CLI uses a hierarchical configuration system (highest priority first):

<FlowDiagram
  title="Configuration Priority"
  caption="Higher priority sources override lower ones"
  direction="vertical"
>
  <FlowNode
    title="CLI Flags"
    icon="terminal"
    variant="primary"
  >
    --url, --key, --deployment (highest priority)
  </FlowNode>
  <FlowNode
    title="Environment Variables"
    icon="settings"
    variant="warning"
  >
    CONVEX_URL, CONVEX_DEPLOY_KEY
  </FlowNode>
  <FlowNode
    title="Project Config"
    icon="file"
    variant="default"
  >
    ./memoir.config.json (optional)
  </FlowNode>
  <FlowNode
    title="User Config"
    icon="user"
    variant="success"
  >
    ~/.memoirrc (managed by CLI, lowest priority)
  </FlowNode>
</FlowDiagram>

### Example: How Priority Works

```bash
# User config (~/.memoirrc) says: local deployment
# Environment variable says: CONVEX_URL=http://127.0.0.1:3210
# CLI flag overrides everything:
memoir db stats --url https://prod.convex.cloud

# Result: Uses production URL from flag
```

## User Configuration (~/.memoirrc)

The CLI automatically manages `~/.memoirrc` for deployment settings.

### Viewing Configuration

<Terminal>memoir config show</Terminal>

<Terminal>memoir config list</Terminal>

<Terminal>memoir config path</Terminal>

### Configuration File Format

The `~/.memoirrc` file is JSON:

```json
{
  "deployments": {
    "local": {
      "url": "http://127.0.0.1:3210",
      "deployment": "anonymous:anonymous-memoir-sdk-local",
      "projectPath": "/Users/you/projects/my-agent",
      "enabled": true
    },
    "staging": {
      "url": "https://staging.convex.cloud",
      "key": "dev:staging|key",
      "projectPath": "/Users/you/projects/my-agent",
      "enabled": false
    },
    "production": {
      "url": "https://prod.convex.cloud",
      "key": "prod:prod|key",
      "projectPath": "/Users/you/projects/my-agent",
      "enabled": false
    }
  },
  "apps": {
    "quickstart": {
      "type": "nextjs",
      "projectPath": "/Users/you/projects/my-agent",
      "path": "quickstart",
      "port": 3000,
      "startCommand": "npm run dev",
      "enabled": true
    }
  },
  "default": "local",
  "format": "table",
  "confirmDangerous": true
}
```

## Managing Deployments

### Adding Deployments

<Steps>
  <Step title="Interactive mode (recommended)">

<Terminal>memoir config add-deployment</Terminal>

  </Step>
  <Step title="With options">

<Terminal>memoir config add-deployment cloud --url https://your-app.convex.cloud --key "prod|..."</Terminal>

  </Step>
  <Step title="Set as default">

<Terminal>memoir config add-deployment production --url https://prod.convex.cloud --default</Terminal>

  </Step>
</Steps>

### Removing Deployments

<Terminal>memoir config remove-deployment</Terminal>

<Terminal>memoir config remove-deployment staging</Terminal>

<Callout type="note">
  Cannot remove the default deployment. Set a different default first.
</Callout>

### Updating Deployment Settings

<Terminal>memoir config set-url production --url https://new-prod.convex.cloud</Terminal>

<Terminal>memoir config set-key production --key "prod|new-key"</Terminal>

<Terminal>memoir config set-path production /path/to/project</Terminal>

### Enabling/Disabling Deployments

Control which deployments start with `memoir start`:

<Terminal>memoir config enable staging</Terminal>

<Terminal>memoir config disable production</Terminal>

<Terminal>memoir config list</Terminal>

### Switching Deployments

<Terminal>memoir use</Terminal>

<Terminal>memoir use production</Terminal>

<Terminal>memoir use --clear</Terminal>

### Testing Configuration

You can verify your configuration by checking the connection:

<Terminal>memoir config show</Terminal>

<Terminal>memoir db stats</Terminal>

## Multi-Environment Setup

### Example: Local + Staging + Production

<Steps>
  <Step title="Add local development">

<Terminal>memoir config add-deployment local --url http://127.0.0.1:3210</Terminal>

<Terminal>memoir config set-path local ~/projects/my-agent</Terminal>

  </Step>
  <Step title="Add staging">

<Terminal>memoir config add-deployment staging --url https://staging.convex.cloud</Terminal>

<Terminal>memoir config set-key staging</Terminal>

  </Step>
  <Step title="Add production">

<Terminal>memoir config add-deployment production --url https://prod.convex.cloud</Terminal>

<Terminal>memoir config set-key production</Terminal>

  </Step>
  <Step title="Enable only local for auto-start">

<Terminal>memoir config enable local</Terminal>

<Terminal>memoir config disable staging</Terminal>

<Terminal>memoir config disable production</Terminal>

<Callout type="tip">
  Now `memoir start` starts only local. Use `memoir start -d staging` or `memoir start -d production` for other environments.
</Callout>

  </Step>
</Steps>

### Switching Between Environments

<Terminal>memoir use local</Terminal>

<Terminal>memoir use production</Terminal>

<Terminal>memoir db stats -d staging</Terminal>

## Project Configuration (Optional)

### memoir.config.json

You can optionally create a `memoir.config.json` in your project root:

```json
{
  "defaultMemorySpace": "my-agent",
  "defaultImportance": 50,
  "memoryRetention": {
    "maxVersions": 10,
    "retentionDays": 365
  },
  "graph": {
    "enabled": true,
    "uri": "bolt://localhost:7687"
  }
}
```

<Callout type="note">
  This is optional. Most configuration is managed via CLI in `~/.memoirrc`.
</Callout>

## SDK Configuration

When using the SDK programmatically, configuration is minimal:

```typescript
import { Memoir } from "@getmemoir/sdk";

// Minimal configuration
const memoir = new Memoir({
  convexUrl: process.env.CONVEX_URL!, // Set by CLI or .env.local
});
```

### With Graph Database

```typescript
import {
  CypherGraphAdapter,
  initializeGraphSchema,
} from "@getmemoir/sdk/graph";

const graphAdapter = new CypherGraphAdapter();
await graphAdapter.connect({
  uri: process.env.NEO4J_URI!,
  username: process.env.NEO4J_USERNAME!,
  password: process.env.NEO4J_PASSWORD!,
});

await initializeGraphSchema(graphAdapter);

const memoir = new Memoir({
  convexUrl: process.env.CONVEX_URL!,
  graph: {
    adapter: graphAdapter,
    orphanCleanup: true,
  },
});
```

## Best Practices

1. **Use environment variables** - Never hardcode URLs or keys
2. **Separate environments** - Use `memoir use local|staging|production`
3. **Secure credentials** - Use `memoir config set-key` (prompts securely), add `.env.local` to `.gitignore`
4. **Test before production** - Always verify on staging first
5. **Regular backups** - Run `memoir db backup` before major changes
