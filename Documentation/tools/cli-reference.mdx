# Memoir CLI Reference

<Callout type="info">

**Last Updated**: 2026-01-08 | **CLI Version**: 0.27.4 | **Package**: `@getmemoir/cli`

</Callout>

## What is Memoir CLI?

The Memoir CLI is a command-line tool for managing Memoir deployments, performing administrative tasks, and streamlining development workflows. It provides comprehensive functionality for:

- **Project initialization** and lifecycle management
- **Database operations** and statistics
- **Memory management** (vector store operations)
- **User data management** with GDPR compliance
- **Deployment management** across multiple environments
- **Interactive development** with live-reloading

## Installation

Install via Homebrew (Recommended):

<Terminal>brew install memoir/tap/cli</Terminal>

Or install globally via npm:

<Terminal>npm install -g @getmemoir/cli</Terminal>

Or use with npx (no installation needed):

<Terminal>npx @getmemoir/cli &lt;command&gt;</Terminal>

Or install as dev dependency in your project:

<Terminal>npm install --save-dev @getmemoir/cli</Terminal>

**Requirements**: Node.js >= 20

## Detailed Command Reference

### Project Lifecycle Commands

#### `memoir init`

Initialize a new Memoir project with interactive setup wizard.

**Usage:**

```bash
memoir init [directory] [options]
```

**Arguments:**

- `directory` - Target directory (defaults to current directory)

**Options:**

- `--local` - Quick setup with local Convex only [default: false]
- `--cloud` - Quick setup with cloud Convex only [default: false]
- `--skip-graph` - Skip graph database setup [default: false]
- `-t, --template <name>` - Template to use [default: "basic"]
- `-y, --yes` - Skip confirmation prompts [default: false]
- `--start` - Start Convex dev server after setup [default: false]

**Available Templates:**

<DataTable
  headers={["Template", "Description"]}
  rows={[
    ["`basic`", "Interactive CLI + HTTP API server (default)"],
    ["`vercel-ai-quickstart`", "Next.js ChatGPT-style web app with Vercel AI SDK"],
    ["`chat-sdk`", "Full-featured chat app based on vercel/ai-chatbot with Memoir"],
  ]}
/>

**Examples:**

```bash
# Interactive setup (recommended)
memoir init

# Create project with local Convex (fastest)
memoir init my-app --local

# Create project with cloud Convex
memoir init my-app --cloud

# Create and start immediately
memoir init my-app --local --start

# Skip graph database setup
memoir init my-app --skip-graph

# Use Chat SDK template (full-featured chat app)
memoir init my-chatbot --template chat-sdk
```

**What It Does:**

1. Creates project directory and copies template files
2. Configures Convex backend (local or cloud)
3. Optionally sets up graph database (Neo4j or Memgraph)
4. Optionally configures OpenAI API key
5. Installs dependencies
6. Deploys Memoir backend functions
7. Saves deployment configuration to `~/.memoirrc`
8. Optionally installs Vercel AI quickstart demo

**Related:**

- [Getting Started: Installation](/getting-started/installation)
- [Getting Started: Five Minute Quickstart](/getting-started/five-minute-quickstart)

#### `memoir start`

Start development services for all enabled deployments.

**Usage:**

```bash
memoir start [options]
```

**Options:**

- `-d, --deployment <name>` - Start a specific deployment only
- `-l, --local` - Use Convex local beta mode (starts a new local backend) [default: false]
- `-f, --foreground` - Run in foreground (only works with single deployment) [default: false]
- `--convex-only` - Only start Convex servers [default: false]
- `--graph-only` - Only start graph databases [default: false]

**Examples:**

```bash
# Start all enabled deployments
memoir start

# Start specific deployment
memoir start --deployment production

# Start with local Convex
memoir start --local

# Start in foreground (see logs)
memoir start --deployment local --foreground

# Start only graph databases
memoir start --graph-only
```

**What It Does:**

1. Reads deployment configuration from `~/.memoirrc`
2. For each enabled deployment:
   - Starts graph database if configured
   - Deploys functions to production (for cloud deployments)
   - Starts Convex dev server in background
3. Starts enabled template apps
4. Saves process IDs for later management

**Related:**

- [Configuration: Managing Deployments](#memoir-config)
- [`memoir stop`](#memoir-stop)
- [`memoir dev`](#memoir-dev)

#### `memoir stop`

Stop background services (Convex, graph database, and apps).

**Usage:**

```bash
memoir stop [options]
```

**Options:**

- `-d, --deployment <name>` - Stop specific deployment only
- `-a, --app <name>` - Stop specific app only
- `--convex-only` - Only stop Convex server [default: false]
- `--graph-only` - Only stop graph database [default: false]
- `--apps-only` - Only stop template apps [default: false]

**Examples:**

```bash
# Stop all services
memoir stop

# Stop specific deployment
memoir stop --deployment local

# Stop specific app
memoir stop --app quickstart

# Stop only Convex (leave graph running)
memoir stop --convex-only
```

**What It Does:**

1. Finds running processes using PID files
2. Sends SIGTERM to Convex dev servers
3. Stops Docker containers for graph databases
4. Stops template apps
5. Cleans up PID files

**Related:**

- [`memoir start`](#memoir-start)
- [`memoir status`](#memoir-status)

#### `memoir dev`

Start interactive development mode with Expo-style interface.

**Usage:**

```bash
memoir dev [options]
```

**Options:**

- `-d, --deployment <name>` - Run specific deployment only
- `-l, --local` - Force local Convex instance for all deployments [default: false]

**Keyboard Shortcuts:**

- `c` - Clear screen and show status
- `s` - Show status dashboard
- `r` - Restart all services
- `g` - Toggle graph database
- `k` - Kill Convex instances (port conflicts)
- `q` - Quit (or Ctrl+C)
- `?` or `h` - Show help

**Examples:**

```bash
# Start interactive dev mode
memoir dev

# Dev mode for specific deployment
memoir dev --deployment local

# Force local mode
memoir dev --local
```

**What It Does:**

1. Syncs schema files from SDK
2. Starts all enabled deployments and apps
3. Displays live status dashboard
4. Streams aggregated logs from all services
5. Provides keyboard shortcuts for common actions
6. Handles graceful shutdown on exit

**Features:**

- Multi-deployment support
- Live status updates
- Aggregated streaming logs
- Keyboard shortcuts
- Automatic schema sync
- Graph database management

**Related:**

- [`memoir start`](#memoir-start)
- [`memoir status`](#memoir-status)

##### `memoir config add-deployment`

Add a new deployment configuration.

**Usage:**

```bash
memoir config add-deployment [name] [options]
```

**Options:**

- `-u, --url <url>` - Convex deployment URL
- `-k, --key <key>` - Convex deploy key
- `--default` - Set as default deployment [default: false]
- `--json-only` - Only save to ~/.memoirrc (skip .env.local) [default: false]

**Examples:**

```bash

memoir config add-deployment cloud \
  --url https://my-app.convex.cloud \
  --key "prod|abc123..."

# Add and set as default
memoir config add-deployment staging \
  --url https://staging.convex.cloud \
  --default
```

### Memory Commands

#### `memoir memory`

Manage memories (vector store). The memory command has multiple subcommands:

##### `memoir memory list`

List memories in a memory space.

**Usage:**

```bash
memoir memory list [options]
```

**Options:**

- `-d, --deployment <name>` - Target deployment
- `-s, --space <id>` - Memory space ID **(required)**
- `-u, --user <id>` - Filter by user ID
- `-l, --limit <number>` - Maximum number of results [default: 50]
- `-f, --format <format>` - Output format: `table`, `json`, `csv`

**Examples:**

```bash
# List memories in a space
memoir memory list --space agent-1

# Filter by user
memoir memory list --space agent-1 --user user-123

# Increase limit
memoir memory list --space agent-1 --limit 100
```

**Related:**

- [Core Features: Semantic Search](/core-features/semantic-search)

##### `memoir users delete`

Delete user profile with optional GDPR cascade deletion.

**Usage:**

```bash
memoir users delete <userId> [options]
```

**Options:**

- `-d, --deployment <name>` - Target deployment
- `--cascade` - Delete all user data across all layers (GDPR) [default: false]
- `--dry-run` - Preview what would be deleted without actually deleting [default: false]
- `--verify` - Verify deletion completeness after cascade [default: true]
- `-y, --yes` - Skip confirmation prompt (not recommended for cascade) [default: false]

**Examples:**

```bash
# Delete user profile only
memoir users delete user-123

# GDPR cascade deletion (all user data)
memoir users delete user-123 --cascade

# Dry run to preview deletion
memoir users delete user-123 --cascade --dry-run
```

<Callout type="warning">
Cascade deletion permanently removes ALL user data:
</Callout>

- User profile
- Conversations and messages
- Memories (vector store)
- Facts
- Immutable records
- Mutable records
- Graph nodes (if configured)

**Related:**

- [GDPR Compliance](#gdpr-compliance)

## Configuration

The CLI uses a hierarchical configuration system.

### Configuration Priority (highest first)

1. **CLI flags**: `--url`, `--key`, `--deployment`
2. **Environment variables**: `CONVEX_URL`, `CONVEX_DEPLOY_KEY`
3. **Project config**: `./memoir.config.json`
4. **User config**: `~/.memoirrc`

### User Config File

**Location:** `~/.memoirrc`

**Format:** JSON

```json
{
  "deployments": {
    "local": {
      "url": "http://127.0.0.1:3210",
      "deployment": "anonymous:anonymous-memoir-sdk-local",
      "projectPath": "/Users/me/projects/my-memoir-app",
      "enabled": true
    },
    "staging": {
```

## Config Auto-Cleanup

The CLI automatically validates and cleans stale entries from your configuration during normal operations. This prevents outdated deployments or apps from cluttering your config or causing errors.

### When Cleanup Occurs

Validation runs automatically when you use certain commands:

<DataTable
  headers={["Command", "Validation Type", "Description"]}
  rows={[
    ["`memoir status`", "Full", "Checks paths + Convex URL accessibility"],
    ["`memoir config list`", "Full", "Checks paths + Convex URL accessibility"],
    ["`memoir config show`", "Full", "Checks paths + Convex URL accessibility"],
    ["`memoir start`", "Path only", "Checks paths only (faster startup)"],
  ]}
/>

### What Gets Removed

<Callout type="info">
Entries are only removed when there's definitive evidence they're invalid. The CLI errs on the side of caution to prevent accidental data loss.
</Callout>

**Deployments are removed when:**
- The `projectPath` directory no longer exists on your filesystem
- The Convex URL returns an authentication error (401/403)

**Apps are removed when:**
- The combined `projectPath` + `path` directory no longer exists

### Safety Rails

To prevent accidental removal due to transient issues:

<DataTable
  headers={["Scenario", "Action", "Reason"]}
  rows={[
    ["Network error (ECONNREFUSED, etc.)", "Keep", "Server might be temporarily down"],
    ["Timeout (>3s)", "Keep", "Slow network shouldn't delete config"],
    ["DNS failure", "Keep", "Temporary DNS issues"],
    ["Server errors (500, 502, 503, 504)", "Keep", "Backend issues are transient"],
    ["Auth error (401, 403)", "Remove", "Indicates invalid credentials"],
    ["Missing project path", "Remove", "Directory definitively doesn't exist"],
  ]}
/>

### Default Deployment Reassignment

When the default deployment is removed during auto-cleanup:
1. Remaining deployments are sorted alphabetically
2. The first one becomes the new default
3. If no deployments remain, default becomes empty

### Cleanup Notification

When entries are removed, you'll see a notification:

<Terminal>Cleaned 1 stale deployment(s): old-project</Terminal>

Or when both deployments and apps are removed:

<Terminal>Cleaned 2 stale deployment(s), 1 app(s)</Terminal>

### Manual Cleanup

You can also manually remove deployments and apps:

Remove a deployment:

<Terminal>memoir config remove-deployment &lt;name&gt;</Terminal>

View current config to verify cleanup:

<Terminal>memoir config show</Terminal>

## GDPR Compliance

The CLI provides GDPR-compliant data management features.

### Cascade Deletion

When deleting users with the `--cascade` flag, ALL user data is permanently removed:

<Terminal>memoir users delete user-123 --cascade</Terminal>

**What Gets Deleted:**

- User profile (all versions)
- Conversations (all messages)
- Memories (vector store)
- Facts
- Immutable records
- Mutable records
- Graph nodes (if configured)

**Verification:**

Use `--verify` (enabled by default) to check deletion completeness:

<Terminal>memoir users delete user-123 --cascade --verify</Terminal>

**Dry Run:**

Preview what would be deleted without actually deleting:

<Terminal>memoir users delete user-123 --cascade --dry-run</Terminal>

### Data Portability

Export all user data in machine-readable format:

<Terminal>memoir users export user-123 --output user-data.json</Terminal>

**Related:**

- [`memoir users delete`](#memoir-users-delete)
- [`memoir users export`](#memoir-users-export)

## Safety Features

The CLI includes multiple safety features for dangerous operations:

### Confirmation Prompts

Dangerous operations require explicit confirmation:

<Terminal>memoir db clear</Terminal>

This prompts: "Clear ALL data from local?"

Bypass with `--yes` flag (use carefully):

<Terminal>memoir db clear --yes</Terminal>

### Dry Run Mode

Preview what would happen without executing:

Preview user deletion:

<Terminal>memoir users delete user-123 --cascade --dry-run</Terminal>

Preview database restore:

<Terminal>memoir db restore --input backup.json --dry-run</Terminal>

### Exact Confirmation

For cascade deletions, you must type the exact ID:

<Terminal>memoir users delete user-123 --cascade</Terminal>

This prompts: Type "user-123" to confirm cascade deletion.

### Verification

Automatic verification after cascade deletions:

<Terminal>memoir users delete user-123 --cascade --verify</Terminal>

This verifies all user data was deleted and reports any potential issues.

## Troubleshooting

### Common Issues

#### Command Not Found

```
memoir: command not found
```

**Solution:**

Install via Homebrew (Recommended):

<Terminal>brew install memoir/tap/cli</Terminal>

Or install globally via npm:

<Terminal>npm install -g @getmemoir/cli</Terminal>

Or use npx:

<Terminal>npx @getmemoir/cli &lt;command&gt;</Terminal>

---

#### No Deployments Configured

```
No deployments configured
```

**Solution:**

Initialize a new project:

<Terminal>memoir init</Terminal>

Or add existing deployment:

<Terminal>memoir config add-deployment</Terminal>

---

#### Connection Failed

```
Connection failed: ECONNREFUSED
```

**Solutions:**

- Check if Convex is running: `memoir status`
- Start Convex: `memoir start`
- Verify URL: `memoir config show`
- Test connection: `memoir config test`

---

#### PID File Stale

```
Convex was already stopped
```

**Cause:** Process stopped without cleaning up PID file.

**Solution:**

Clean up PID files:

<Terminal>rm .convex-dev.pid .quickstart-app.pid</Terminal>

Or use dev mode and press 'k' to kill processes by port:

<Terminal>memoir dev</Terminal>

---

#### Schema Sync Failed

```
Schema sync failed: SDK not found
```

**Solutions:**

Install SDK:

<Terminal>npm install @getmemoir/sdk</Terminal>

Or skip schema sync:

<Terminal>memoir deploy --skip-sync</Terminal>

---

#### Graph Database Not Starting

```
Graph database configured but Docker not available
```

**Solutions:**

- Start Docker Desktop
- Check Docker: `docker ps`
- Start manually: `docker-compose -f docker-compose.graph.yml up -d`

---

### Debug Mode

Enable debug output for troubleshooting:

Via flag:

<Terminal>memoir &lt;command&gt; --debug</Terminal>

Via environment variable:

<Terminal>DEBUG=1 memoir &lt;command&gt;</Terminal>

**Debug Output Includes:**

- Environment variable loading
- Configuration resolution
- API request/response details
- Process spawning details
- File operations

## Real-World Workflows

### Workflow 1: New Project Setup

<Steps>
  <Step title="Create new project with local Convex">
    <Terminal>memoir init my-app --local</Terminal>
  </Step>
  <Step title="Navigate to project">
    <Terminal>cd my-app</Terminal>
  </Step>
  <Step title="Start development services">
    <Terminal>memoir start</Terminal>
  </Step>
  <Step title="Check status">
    <Terminal>memoir status</Terminal>
  </Step>
  <Step title="View database statistics">
    <Terminal>memoir db stats</Terminal>
  </Step>
</Steps>

---

### Workflow 2: Multi-Environment Setup

<Steps>
  <Step title="Add deployments">
    <Terminal>memoir config add-deployment local --url http://127.0.0.1:3210</Terminal>
    <Terminal>memoir config add-deployment staging --url https://staging.convex.cloud --key "staging|..."</Terminal>
    <Terminal>memoir config add-deployment production --url https://prod.convex.cloud --key "prod|..."</Terminal>
  </Step>
  <Step title="Set project paths">
    <Terminal>memoir config set-path local /path/to/project</Terminal>
    <Terminal>memoir config set-path staging /path/to/project</Terminal>
    <Terminal>memoir config set-path production /path/to/project</Terminal>
  </Step>
  <Step title="Enable only local for auto-start">
    <Terminal>memoir config enable local</Terminal>
    <Terminal>memoir config disable staging</Terminal>
    <Terminal>memoir config disable production</Terminal>
  </Step>
  <Step title="Deploy to production">
    <Terminal>memoir deploy --deployment production</Terminal>
  </Step>
  <Step title="View all deployments">
    <Terminal>memoir config list</Terminal>
  </Step>
</Steps>

---

### Workflow 3: GDPR Data Export & Deletion

<Steps>
  <Step title="Search for user">
    <Terminal>memoir users list | grep john@example.com</Terminal>
  </Step>
  <Step title="View user data">
    <Terminal>memoir users stats user-123</Terminal>
  </Step>
  <Step title="Export user data (GDPR portability)">
    <Terminal>memoir users export user-123 --output user-123-export.json</Terminal>
  </Step>
  <Step title="Preview deletion">
    <Terminal>memoir users delete user-123 --cascade --dry-run</Terminal>
  </Step>
  <Step title="Delete user data (GDPR right to erasure)">
    <Terminal>memoir users delete user-123 --cascade --verify</Terminal>
  </Step>
</Steps>

---

### Workflow 4: Interactive Development

<Steps>
  <Step title="Start interactive dev mode">
    <Terminal>memoir dev</Terminal>
    
    **Keyboard shortcuts:**
    - `c` - clear screen
    - `s` - show status
    - `r` - restart services
    - `g` - toggle graph database
    - `k` - kill stuck processes
    - `q` - quit
  </Step>
  <Step title="Watch logs in real-time">
    Logs are automatically displayed in the terminal.
  </Step>
  <Step title="Make code changes">
    Convex auto-reloads when you save files.
  </Step>
  <Step title="Toggle graph database">
    Press `g` key.
  </Step>
  <Step title="Quit gracefully">
    Press `q` key.
  </Step>
</Steps>

  </Step>

## Examples

### Example 1: Search and Clear Memories

Search for sensitive data:

<Terminal>memoir memory search "password" --space agent-1</Terminal>

Review results and delete specific memory:

<Terminal>memoir memory delete mem-456 --space agent-1</Terminal>

Or clear all for a user:

<Terminal>memoir memory clear --space agent-1 --user user-123</Terminal>

---

### Example 2: Manage Memory Spaces

Create team space:

<Terminal>memoir spaces create team-alpha --type team --name "Team Alpha"</Terminal>

Add participants:

<Terminal>memoir spaces add-participant team-alpha --id user-123 --type user</Terminal>
<Terminal>memoir spaces add-participant team-alpha --id agent-1 --type ai-agent</Terminal>

View statistics:

<Terminal>memoir spaces stats team-alpha</Terminal>

List participants:

<Terminal>memoir spaces participants team-alpha</Terminal>

Archive when done:

<Terminal>memoir spaces archive team-alpha --reason "Project completed"</Terminal>

---

### Example 3: Export Conversations

List conversations:

<Terminal>memoir conversations list --space agent-1 --user user-123</Terminal>

Get specific conversation:

<Terminal>memoir conversations get conv-789</Terminal>

Export to file:

<Terminal>memoir conversations export conv-789 --format txt --output conv.txt</Terminal>

---

### Example 4: Database Statistics

View overall stats:

<Terminal>memoir db stats</Terminal>

View space-specific stats:

<Terminal>memoir spaces stats agent-1</Terminal>
<Terminal>memoir memory stats --space agent-1</Terminal>

View user-specific stats:

<Terminal>memoir users stats user-123</Terminal>

Export to JSON for analysis:

<Terminal>memoir db stats --format json &gt; db-stats.json</Terminal>
